## 二、项目背景
协鑫集成经营计划部智能平台，目标是建设一个统一的大模型应用平台，逐步覆盖：
- 物流管理
- 计划 BOM
- 经营分析
- 物料管理

后续再扩展：
- 文档知识库 / RAG
- 工具层
- Agent 编排层

当前阶段只重点推进：
**物流一期**。

## 三、物流一期当前业务范围

2. 正式系统数据：2026 年后 MySQL

- “运量”默认按瓦数口径（`shipment_watt`）

### 当前不作为可靠能力的部分

## 七、当前项目的关键判断

### 1. 现在不适合继续无止境做后端版本号堆叠

- 把已有后端能力展示出来

1. 物流一期收口
2. 物流文档知识库 / RAG

5. Agent 层

## 八、当前已知问题 / 注意事项

### 1. 不能假设历史补丁已合入
每次必须以当前仓库代码为准重新审查。

### 2. 不要把 fallback 当成正式业务结果来源
尤其不能返回误导业务语义的值。

### 3. 当前 SQL preview 主要用于解释，不是数据库真实执行 SQL。

### 4. 项目交付时不要带：
- `.env`
- `__pycache__`
- `*.pyc`
- `.idea`
- `.pytest_cache`
- `__MACOSX`
