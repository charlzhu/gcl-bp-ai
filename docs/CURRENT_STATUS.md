# CURRENT_STATUS.md

## 一、项目概述

项目名称：协鑫集成经营计划部智能平台  
当前优先域：物流一期  
当前目标：先把物流一期做成“可试运行、可联调、可给业务试用”的版本，再逐步扩展到计划 BOM、经营分析、物料管理，以及后续工具层和 Agent 层。

---

## 二、当前业务边界

### 1. 当前物流一期数据范围
- 历史数据：2023–2025 年 Excel
- 正式系统数据：2026 年后的 MySQL 业务数据
- 2026 年前系统库中的记录视为测试数据，不纳入正式统计与业务分析

### 2. 当前一期能力边界
当前一期重点不是纯文档 RAG，而是：
- 历史 Excel 标准化
- 2026+ 系统数据同步
- 统一查询服务
- 自然语言查询入口
- 前端查询页

### 3. 当前明确不作为一期可靠能力的内容
- 仓库维度精确统计（当前采用路线 1：不补 allocate 链路）
- 多域真正执行（当前以 logistics 域为主）
- 复杂 Agent 编排
- 高风险业务动作自动执行

---

## 三、当前后端状态

### 1. 数据层状态
已具备：
- 历史 Excel 导入 ETL
- 2026+ 系统数据同步
- ODS / DWD / DWS / DM 分层
- 本地中间库可支撑物流一期查询

### 2. 查询服务状态
统一查询服务已演进至 V10 收口阶段，已完成的主要能力包括：
- V2：真实问题集规则增强
- V3：规则 + 配置化问法模板
- V4：同义词词典 + 指标字典 + 枚举映射
- V5：多域模板分层 + 配置热更新 + 命中解释
- V6：配置版本管理 + 模板优先级评分 + 冲突消解
- V7：执行器接入 + SQL 模板绑定 + 查询计划落库
- V8：参数安全校验 + SQL 白名单 + 执行审计增强
- V9：结果解释增强 + 无结果原因分析 + 执行异常兜底
- V10：稳定版收口（错误码、状态标准化、业务编号探测等）

### 3. 当前已验证通过的关键链路
- aggregate：月度运量 / 费用 / 统计类问题
- detail：合同编号、SAP 单号、发货指令等明细类问题
- compare：跨年同月对比类问题
- 自然语言语义理解：
  - 承运商 / 物流公司
  - 发货量 / 运量 / 出货量
  - 铁运 / 铁路 / 公路 / 汽运
  - 历史 / 系统 / 混合来源

### 4. 当前已知但不一定全部合入的修复补丁
以下补丁曾被单独做过，但不能假设它们已在当前仓库中，必须由 Codex 先以当前代码检查：
- V10 空结果误判修复补丁
- compare 空结果误判 + fallback 承运商映射错误修复补丁

   - 关键改动说明
   - 测试方式
   - 风险点
