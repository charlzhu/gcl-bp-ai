## 一、任务名称
前端查询页第二版优化（基于当前后端接口）

---

## 二、开始前必须做的事情

### 1. 先审查当前仓库状态
必须先确认：
- 当前前端 V1 是否已完整存在
- 当前后端返回结构是否与前端 V1 实际对接
- compare 修复是否已经在当前后端仓库中
- fallback 承运商映射修复是否已经在当前后端仓库中

### 2. 输出一份当前判断
至少说明：
- 当前可直接复用的页面有哪些
- 当前缺哪些页面
- 是否存在前端 V2 需要依赖的后端阻塞项

在做完这一步前，不要直接开始写代码。

---

## 三、本次要完成的目标

### 前端 V2 目标
在当前前端第一版基础上，完成以下增强：

1. **结果表格增强**
   - 列显示更稳定
   - 空值展示统一
   - 数值格式化
   - 结果展示更适合业务查看

2. **明细视图**
   - 从查询结果进入明细页
   - 明细页支持字段完整展示
   - 能区分来源（hist / sys）
   - 便于联调 detail 场景

3. **查询历史**
   - 能查看历史问题
   - 能查看解析结果 / 执行模式 / 状态
   - 能查看当前一次查询的完整返回内容

---

## 四、开发要求

### 代码要求
- 所有新增和修改代码都必须写中文注释
- 保持当前 Vue3 + Element Plus 风格一致
- 尽量做增量修改，不重写整套前端
- 不要引入与当前项目不匹配的重型 UI 依赖

### 业务要求
- 优先服务物流一期
- 不要为了多域预留而把前端复杂度做太高
- 当前界面优先“能联调、能演示、能查问题”，不是追求最终视觉稿

---

## 五、如果发现后端阻塞

如果在实现前端 V2 过程中发现以下问题仍存在：
- compare 场景解释仍然错误
- fallback 结果仍有业务语义误导
- 查询历史接口或数据结构不足以支撑前端展示

则处理原则是：

1. 先记录问题
2. 明确指出阻塞点
3. 只做最小必要后端修复
4. 不要顺手把后端又扩成一个大版本
